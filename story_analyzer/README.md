# Анализатор библиотеки эротических рассказов

Скрипт сканирует папку с txt-файлами (в т.ч. по категориям — подпапкам), отправляет текст в локальную LLM (Ollama, модель `dolphin-mistral`) и сохраняет оценки в **отдельный CSV в той же папке**, где лежат рассказы. В каждой категории — свой `analysis_results.csv`.

## Структура папок

```
input_stories/
  категория_1/
    story1.txt
    story2.txt
    ...
    analysis_results.csv   ← создаётся скриптом
  категория_2/
    ...
    analysis_results.csv
  ...
```

То есть скрипт **кладёт файл результатов в ту же папку, откуда брал рассказы**.

## Поле Summary (описание для аудио)

В CSV есть колонка **Summary** — краткое изложение сюжета на 5–7 предложений (завязка, развитие, кульминация). Оно предназначено для описания аудиорассказа при публикации. В консоли выводится превью этого описания после оценок.

## Требования

- Python 3.10+
- [Ollama](https://ollama.com/) установлен и запущен
- Модель загружена: `ollama pull dolphin-mistral`

## Установка

```bash
cd story_analyzer
pip install -r requirements.txt
```

## Использование

1. Разложите txt-файлы по папкам (категориям) внутри `input_stories`. Можно также положить txt в корень `input_stories`.
2. Запустите:

```bash
python analyze_stories.py
```

3. В каждой папке с рассказами появится свой `analysis_results.csv` с колонками: Filename, Erotic_Score, Audio_Score, Grammar_Score, **Summary**, Tags, Verdict.

В консоли выводятся: имя файла, оценки E/A/G и начало описания (Summary). При битом JSON от модели выполняется попытка восстановления; при неудаче файл пропускается.

## Параметры (в коде)

- `MAX_TEXT_LENGTH` — обрезка длинных текстов (по умолчанию 12000 символов).
- `TEMPERATURE` — 0.3 для стабильных оценок.
- `OLLAMA_URL` — по умолчанию `http://localhost:11434/api/chat`.
- `CSV_FILENAME` — имя файла результатов в каждой папке (`analysis_results.csv`).
